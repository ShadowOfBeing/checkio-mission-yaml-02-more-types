Это вторая задача по парсингу YAML. Она представляет собой следующий этап усложнения парсинга, в котором добавляются типы данных, такие как null и bool, а также возможность использовать кавычки в строках.

Вот несколько примеров:

name: "Bob Dylan"
children: 6

{
  "name": "Bob Dylan", 
  "children": 6
}

name: "Alex \\"Fox\\""
children: 6

{
  "name": 'Alex "Fox"',
  "children": 6
}

Как видите, строку можно задавать в кавычках. Это могут быть как двойные, так и одинарные кавычки.

name: "Bob Dylan"
children: 6
alive: false

{
  "name": "Bob Dylan", 
  "alive": {% if interpreter.slug == "js-node" %}false{% else %}False{%endif%}, 
  "children": 6
}

true и false - это ключевые слова, определяющие булевный тип.

name: "Bob Dylan"
children: 6
coding:

{
  "coding": {% if interpreter.slug == "js-node" %}null{% else %}None{%endif%}, 
  "name": "Bob Dylan", 
  "children": 6
}

Если значение не указано, то оно становится неопределенным. Также для этого есть ключевое слово null.

Входные данные: Строка в формате

Выходные данные: Объект.
